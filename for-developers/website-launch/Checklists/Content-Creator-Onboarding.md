# Content Creator Onboarding Checklist

## Overview

Use this checklist to onboard new content creators and ensure they're confident using SkyCMS to manage your website content.

### Purpose

This checklist helps:
- **New Team Members**: Quickly get up to speed
- **Managers**: Track onboarding progress
- **Administrators**: Ensure consistency across team

### Who Uses This

- **New Content Creators**: Everyone joining the content team
- **New Editors**: Anyone with content management responsibilities
- **Contributing Team Members**: Subject matter experts needing to create content

### Timeline

**Complete within**: First week of employment or content responsibilities

**Recommended pace:**
- **Day 1**: Access setup, system orientation
- **Day 2-3**: Training on core tasks
- **Day 4-5**: Practice and first live content
- **Week 2**: Follow-up and advanced topics

---

## Pre-Onboarding (Before First Day)

### Account Setup

- [ ] **Email created**: New team member has company email
- [ ] **SkyCMS account created**: Username/email established
- [ ] **Password set**: Temporary password generated
- [ ] **Role assigned**: Correct permissions set (Editor, Contributor, etc.)
- [ ] **Access tested**: Administrator verified account works
- [ ] **Welcome email sent**: Contains login info and first-day instructions
- [ ] **Resources prepared**: Guides and documentation compiled
- [ ] **Equipment ready**: Computer/device set up with browser

### Documentation Prepared

- [ ] **User Guide**: Printed or PDF copy
- [ ] **Quick Reference Card**: Common tasks cheat sheet
- [ ] **Content Guidelines**: Brand voice, formatting standards
- [ ] **Contact List**: Support contacts and resources
- [ ] **First Task Prepared**: Initial content to create/edit ready

---

## Day 1: Welcome & System Orientation

### Welcome Session (30 minutes)

Conduct welcome meeting with new person:

- [ ] **Welcome**: Warmly greet team member
- [ ] **Role explained**: Clarify their specific responsibilities
- [ ] **Team introductions**: Introduce to relevant team members
- [ ] **Workspace tour**: Show physical workspace/meeting areas
- [ ] **Tools explained**: Overview of systems they'll use
- [ ] **Questions answered**: Address their questions
- [ ] **Schedule set**: Plan training timeline

### System Access (30 minutes)

Help them log in and explore:

- [ ] **First login**: Successfully log into SkyCMS
- [ ] **Password changed**: Temporary password updated to permanent
- [ ] **Dashboard tour**: Show main interface/navigation
- [ ] **Account settings**: Explain where to find preferences
- [ ] **Help resources**: Point out documentation/support options
- [ ] **Profile updated**: Upload profile photo (optional)

### Documentation Review (30 minutes)

Go over key materials:

- [ ] **User Guide reviewed**: Walk through main sections together
- [ ] **Quick reference distributed**: Explain common tasks card
- [ ] **Content Guidelines reviewed**: Discuss brand voice and standards
- [ ] **Contact list reviewed**: Explain who to contact for what
- [ ] **Q&A completed**: Answer questions about documentation

### Initial Setup

- [ ] **Phone/computer set up**: Tech equipment working
- [ ] **Email functional**: Can send/receive emails
- [ ] **VPN access** (if applicable): Can access systems remotely
- [ ] **Passwords secured**: Credentials stored securely
- [ ] **Two-factor auth** (if used): Set up security features

---

## Day 2-3: Training on Core Tasks

### Training Session 1: Creating Pages (60 minutes)

Hands-on training on page creation:

**Preparation:**
- [ ] Test pages ready for demonstration
- [ ] Example content available
- [ ] Q&A whiteboard ready

**Training:**
- [ ] **Demo first** (20 mins): Show page creation step-by-step
  - [ ] Navigate to Create Page
  - [ ] Enter URL
  - [ ] Select template
  - [ ] Click Create

- [ ] **Guided practice** (25 mins): They create page with help
  - [ ] Create test page together
  - [ ] Show Visual Editor
  - [ ] Add content to editable regions

- [ ] **Q&A** (15 mins): Answer questions
  - [ ] Why certain URLs are used
  - [ ] When to use which template
  - [ ] Common issues

**Deliverable:**
- [ ] New person creates one test page independently

### Training Session 2: Editing Content (60 minutes)

Learn to edit and format content:

**Preparation:**
- [ ] Sample page with content ready to edit
- [ ] Visual Editor interface explained

**Training:**
- [ ] **Demo first** (20 mins): Show editing process
  - [ ] Click Edit button
  - [ ] Enter Visual Editor
  - [ ] Click content areas to edit
  - [ ] Add/remove text

- [ ] **Guided practice** (25 mins): They edit sample page
  - [ ] Edit existing content
  - [ ] Add formatting (bold, italic)
  - [ ] Add links
  - [ ] Practice with heading levels

- [ ] **Q&A** (15 mins): Clarify concepts
  - [ ] Difference between Draft and Published
  - [ ] When changes save automatically
  - [ ] How to undo changes

**Deliverable:**
- [ ] New person edits existing content independently

### Training Session 3: Publishing & Workflow (45 minutes)

Understand the publishing process:

**Preparation:**
- [ ] Workflow diagram prepared
- [ ] Example showing Draft/Published states

**Training:**
- [ ] **Draft vs. Published explained** (15 mins)
  - [ ] Draft = work in progress
  - [ ] Published = live on website
  - [ ] Only published visible to public

- [ ] **Publishing process** (15 mins)
  - [ ] Save changes
  - [ ] Exit Edit mode
  - [ ] Click Publish
  - [ ] Verify live

- [ ] **Workflow review** (15 mins)
  - [ ] Who approves content
  - [ ] When to publish
  - [ ] How to request help
  - [ ] Escalation process

**Deliverable:**
- [ ] New person publishes a test page

## Day 4-5: Hands-On Practice & First Live Content

### Practice Assignment 1: Create New Page

**Task:**
- Create a real page (with supervisor approval on content)

**Checklist:**
- [ ] Page created with correct URL
- [ ] Appropriate template selected
- [ ] Title and meta description added
- [ ] Content added to all editable regions
- [ ] Images/media added (if applicable)
- [ ] Links verified
- [ ] Saved successfully

**Review:**
- [ ] Supervisor reviews draft
- [ ] Feedback provided
- [ ] Revisions made (if needed)
- [ ] Published with approval

### Practice Assignment 2: Edit Existing Page

**Task:**
- Update an existing page with new content

**Checklist:**
- [ ] Page opened in Edit mode
- [ ] Content updated correctly
- [ ] Formatting applied (headings, bullets)
- [ ] Images updated (if applicable)
- [ ] No typos or errors
- [ ] Links verified
- [ ] Saved and published

**Review:**
- [ ] Supervisor reviews changes
- [ ] Verify page displays correctly
- [ ] Provide feedback

### First Live Content (Optional)

- [ ] **Simple content assigned**: Non-critical page/section
- [ ] **Supervisor reviews**: Checks before publication
- [ ] **Published together**: Walk through publishing together
- [ ] **Verified live**: Check published version
- [ ] **Celebrated**: Acknowledge first live content!

## Week 2: Advanced Training & Follow-Up

### Check-In Meeting (30 minutes)

After first week:

- [ ] **How's it going?**: Ask about their experience
- [ ] **Any blockers?**: Identify challenges
- [ ] **Questions?**: Address outstanding questions
- [ ] **Confidence level**: Gauge comfort and independence
- [ ] **Feedback**: Ask for suggestions on training
- [ ] **Next steps**: Discuss advanced training or new tasks

### Advanced Topics (as needed)

Schedule additional training if needed:

- [ ] **Images & Media**: Uploading, optimization, File Manager
  - Time: 30 minutes
  - Format: Hands-on demo

- [ ] **SEO Best Practices**: Titles, descriptions, headings
  - Time: 30 minutes
  - Format: Presentation + review of their pages

- [ ] **Troubleshooting**: Common issues and solutions
  - Time: 30 minutes
  - Format: Walkthrough of common problems

- [ ] **Advanced Formatting**: Tables, embeds, custom HTML
  - Time: 30 minutes
  - Format: Demo + hands-on practice

### One-on-One Support

- [ ] **Office hours**: Schedule regular check-ins (weekly first month)
- [ ] **Slack channel**: For quick questions
- [ ] **Email support**: For detailed issues
- [ ] **Pair programming**: Sit together for complex tasks

## Knowledge Verification

### Understanding Assessment

Verify new person understands key concepts:

**Page Creation:**
- [ ] What is a URL? Can explain purpose
- [ ] What is a template? Can explain when to use different templates
- [ ] What is an editable region? Understand why some content can't be changed

**Editing:**
- [ ] How to enter Edit mode
- [ ] How to click and edit content
- [ ] How to add formatting (bold, italic)
- [ ] How to save work

**Publishing:**
- [ ] Difference between Draft and Published
- [ ] How to publish a page
- [ ] Understand approval workflow
- [ ] Know who to contact with issues

**SEO:**
- [ ] What is a page title? Why it matters
- [ ] What is meta description? Why it matters
- [ ] What is heading hierarchy? Basic structure
- [ ] What is alt text? Why images need it

### Practical Skills Assessment

They should be able to independently:

- [ ] **Create a new page**: Start to finish without help
- [ ] **Edit content**: Find and update text correctly
- [ ] **Publish a page**: Move from Draft to Published
- [ ] **Find help**: Know where to look for resources
- [ ] **Ask questions**: Comfortable reaching out when stuck
