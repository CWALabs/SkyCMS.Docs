# Documentation Standards Rollout Plan (30-60-90)

This rollout plan applies the Documentation Standards Stack v1 in measured phases.

## Days 1-30

- Publish and approve standards stack and templates.
- Add required metadata to high-traffic pages first.
- Add intent-first entry links from main landing pages.
- Add PR checklist usage to docs workflow.

Exit criteria:

- Standards and templates are published.
- New docs use the new templates by default.

## Days 31-60

- Retrofit top 25 high-traffic pages to the new structure.
- Improve related links and cross-navigation.
- Add search synonym mappings from query logs.
- Reduce ambiguous titles in key sections.

Exit criteria:

- Top 25 pages are template-aligned.
- Priority navigation labels have been normalized.

## Days 61-90

- Run tree-testing on top user journeys.
- Use findings to refine navigation labels and grouping.
- Publish first metrics report.
- Create the next prioritized backlog from analytics.

Exit criteria:

- First optimization cycle is completed.
- Baseline metrics are available and tracked.

## KPI targets for first cycle

- 30 percent reduction in top 20 failed search queries.
- 20 percent improvement in first-click success for key tasks.
- 100 percent owner and review-date coverage for active pages.
