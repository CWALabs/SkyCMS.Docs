# Editor Workflow Reference

## New here?

Start with [For Editors](./index.md) for the role overview and quick links, then use this page to map tasks to tools.

## Summary

Editors run day-to-day content operations: drafting, reviewing, publishing, scheduling, and content retirement.

Use this page to map editorial tasks to the right tools and handoff boundaries.

## Editorial workflow

1. Create an article or page draft.
2. Revise and preview content.
3. Request review where workflow requires it.
4. Publish immediately or schedule release.
5. Update and republish as needed.
6. Unpublish or retire obsolete content.

## Verification

Editorial readiness is established when you can:

- create and update drafts,
- preview and publish content,
- confirm live output after publication,
- follow role-based approval constraints correctly.
