# Storage: S3-Compatible

## Summary

Common compatible providers include AWS S3 and other S3-compatible object storage services.

## Required settings

- bucket name,
- region,
- access key ID,
- secret access key,
- optional endpoint override for non-AWS providers.

## Example configuration

Use environment variables and secret stores for credentials. Keep keys scoped to required bucket operations only.

## Validation checklist

- upload, read, and delete operations succeed,
- publish output is written to expected bucket paths,
- region and endpoint values match the active provider,
- IAM and bucket policies permit required operations.

## Common pitfalls

- wrong region or endpoint causing auth or signature errors,
- insufficient IAM rights to read, write, or delete objects,
- bucket policy blocking server-side operations.
